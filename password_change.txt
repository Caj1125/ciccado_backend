Password Change Feature – Backend Implementation Instructions

Objective
Add a secure "Change Password" feature that allows authenticated users to update their account password.

1. Create a new API endpoint
Add a protected route in the authentication routes module.

Endpoint:
PUT /api/auth/change-password

This route must require authentication using the existing JWT authentication middleware.

2. Require authentication
Before the password change logic executes, the request must pass through the JWT authentication middleware.

The middleware should:
- Read the token from the Authorization header
- Verify the token
- Extract the user ID
- Attach the user information to the request object

3. Define request body format
The frontend must send the following fields in the request body:

currentPassword
newPassword
confirmPassword

currentPassword is used to verify the user's identity.
newPassword is the desired new password.
confirmPassword is used to ensure the user did not mistype the password.

4. Retrieve the user from the database
Using the user ID obtained from the authentication middleware, retrieve the corresponding user record from the database.

5. Verify the current password
Compare the provided currentPassword with the stored hashed password.

If the passwords do not match:
- Return an error response
- Do not proceed further

6. Validate the new password
Before updating the password, perform validation checks:

- newPassword must match confirmPassword
- newPassword must not be the same as the currentPassword
- newPassword should meet minimum security requirements (recommended minimum length of 8 characters)

If validation fails, return an appropriate error response.

7. Hash the new password
Use the same password hashing method used during registration (for example bcrypt).

The new password must be hashed before storing it in the database.

8. Update the password in the database
Replace the user's existing password with the newly hashed password.

Save the updated user record to the database.

9. Optional security improvement
After a password change, you may invalidate existing login sessions or require the user to log in again.

For a semester project, simply updating the password is sufficient.

10. Send response
Return a success response to the frontend confirming that the password was updated.

Example response message:
"Password changed successfully"

11. Frontend integration
Add a Change Password page in the React frontend.

The page should include a form with three fields:

Current Password
New Password
Confirm Password

When submitted, send a PUT request to:
PUT /api/auth/change-password

Include the JWT token in the Authorization header.

12. Security considerations
Ensure the following security practices are followed:

- Passwords are never stored in plain text
- Password verification uses secure hashing comparison
- The endpoint is protected using authentication middleware
- Password validation rules are enforced before updating
